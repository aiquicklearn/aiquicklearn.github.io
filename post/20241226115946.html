Build Records of Blog Created by Hexo+Github Pages+Next

Posted on 2024-12-26 In Blog Word count in article: 3.2k Reading time ≈ 12 mins.

引言

AI已经在影响着人类未来的发展了，渺小的个人，要么置之不理、假装与己无关，要么一知半解、终被时代潮流抛在后面，要么迎头赶上这趟时代快速列车。而我选择第三个选项。所以，创建此个人博客，以此记录这段旅程。

AI is shaping humanity’s future. I, as an individual, refuse to be left behind. I’m not choosing ignorance or apathy; I’m choosing to engage. This blog records my journey with AI.【Translated by Gemini which is so good at translating】

我并不懂代码技术，所以搜索了很多适合小白的方案后，经过对比，选择了Github Pages、Markdown、Hexo和NexT来建站，图床则用PicGo、Typora这个组合，主要原因如下：

  • Github支持免费的静态页面托管，网站部署和发布都很简单，小白也可上手。不用关心服务器等资源投入
  • Hexo是一个开源的、支持Markdown语言、受网友欢迎的博客框架，一直在持续更新，支持各种插件、主题，参与贡献的网友众多。
  • nexT是Hexo框架下的一个主题，搜索时发现多数网友都在推荐，且持续更新中。
  • PicGo+Typora可以实现写文章添加图片时，可将图片自动从本地push到github、并将文章中的图片路径替换为网址链接。而Typora本身是个很好用的markdown语言编辑器，所写即所见所得，达到沉浸式写作体验。

本文重点是记录我在搭建个人博客过程中出现的问题，有些问题是搜索了很多帖子才解决的，虽然解决之后再回头看，发现问题都很小，但过程比较磨人，尤其是对于不懂代码的小白来说，所以分享出来。

为了不偏离本网站AI主题，同时建站过程都是通过网络自学完成，为了也践行互联网分享精神，所有建站环节的具体步骤就不一一细致展开了，建议阅读每个环节所列的推荐文章（不用担心，这些文章都经过了我实践检验，正确且可操作😊）。

私有域名 + Cloudflare

是否购买私有域名

如果你并不在意网站域名是否独特，可以忽略此部分内容。

如果你想拥有一个独特域名，可以买一个、或者注册免费域名。

1)购买域名：只要购买后一直付钱，域名就一直是你的。免费域名，平台有倒闭风险。 付费域名网站，对比下来，namecheap是性价比最高、支持域名最全的。有需要可去看看。

虽然“免费的最贵”，但有免费的情况下，还是想试试，怎么办？

2)免费域名：免费网站，很多已经被薅羊毛薅到关门了，剩下可搜索到的几个网站，也不容易注册了。

截止目前，还可以去us.kg这个网站试试，每个用户可以申请3个免费域名。之前被滥用的厉害，于是从24年10月开始，注册要求KYC。如果注册邮箱符合要求、或有github账号，可以免KYC。

私有域名托管到Cloudflare

有了自己的域名后，可以把域名托管到Cloudflare，而us.kg支持cloudflare托管。

us.kg注册 + cloudflare绑定域名，具体步骤可参考下方文章：

  • 可以免费申请的域名us.kg,从域名注册到托管到Cloudflare全流程保姆教程

搜索后，发现比较多网友都是在命令前加上sudo来解决，但这个方法有点不妥，最后找到Stackoverflow论坛的一篇帖子，参考了第二楼回复的方案👇：

Error: EACCES: permission denied, access ‘/usr/local/lib/node_modules’

打开终端后，直接按以上5个步骤，输入对应命令即可。在第3步，需要手工创建文件、打开文件，直接输入下图中其他网友补充的命令行touch .profile,open .profile：

以上方案操作后，重新执行Hexo安装命令，成功🎉

上图可以看到，Hexo安装成功后，执行hexo命令，可能会找不到，可以在命令前都加上npx

安装Hexo主题-NexT

打算用Homebrew来安装NexT（发现都推荐Homebrew在macos安装软件），首先安装Homebrew。遇到了以下问题1-4。

问题1-Github的DNS被污染，无法连接

执行安装Homebrew的指令时，会因网络不能直接访问github报错⬇️：
报错：【curl: (7) Failed to connect to raw.githubusercontent.com port 443: Connection refused】

搜索后，发现是Github的DNS存在污染，需要手动修改电脑本地的hosts👇：

  • Github网友提供的解决方案

如果不知道怎么通过终端修改hosts，可以点击查看这个修改方法👇：

  • mac修改hosts檔案超輕鬆編輯2種實用技巧】

问题2-Github的DNS被污染，无法连接

问题1解决后，接着，又遇到2个报错：

报错1：【error: RPC failed; curl 92 HTTP/2 stream 0 was not closed cleanly: PROTOCOL_ERROR (err 1)】

报错2:【PC failed; curl 18 transfer closed with outstanding read data remaining error: 7721 bytes of body are still expected】

接着找解决方法，在Stackflow论坛找到下方帖子👇：

  • error: RPC failed; curl 92 HTTP/2 stream 0 was not closed cleanly: PROTOCOL_ERROR (err 1)

用了帖子中的这个建议⬇️：

打开终端，执行上述2条命令后，git config --global http.version HTTP/1.1，git config --global http.version HTTP/2，效果⬇️：

问题3-缺少Xcrun的Command Line Tools

继续执行命令，接着又报错，缺少【xcrun: error: invalid active developer path (/Library/Developer/CommandLineTools), missing xcrun at：/……/xcrun】

解决方案大部分都建议安装Xcode，但Xcode太大了（几个G），继续搜，发现可以只安装Command Line Tools（实际也要1个G左右），按以下文章介绍的方法安装

  • MacOS升級 missing xcrun

问题4-还是无法安装Homebrew

解决上述问题后，还是无法安装Homebrew，看终端报错，一些包下载特别慢，最后整个安装超时断掉了，得从头再来。

但其实网络正常的，在耗费很多时间后，决定放弃Homebrew，回到NexT官网，尝试用git。

为什么之前不用git？因为不会用🤦，所以最开始安装Hexo时，按前述的YouTube视频选择了GitHub Desktop。

但在解决本章的问题2时，发现可以用git了，神奇，猜测可能：1）在后续配置网站过程中，因为VS code有提醒安装git，可能误点击了；2）因为装了Command Line Tools）。

于是，打开终端，cd到博客网站文件夹，改用git命令安装NexTgit clone https://github.com/next-theme/hexo-theme-next themes/next，中间报错了一次，但继续执行，最后成功⬇️

💡用git安装的NexT文件夹(next)直接放在【博客网站根目录-themes】下

配置NexT

问题1-NexT取消了支持评论插件Valine

NexT有4个主题可供选择，支持自定义配置。在参考了NexT官方文档、以及各网友文章后，一通配置操作下来，结果在执行npx hexo server时，报错了（可以肯定配置出了问题，因为已多次执行过）：

报错：【TypeError: Cannot read properties of undefined (reading ‘enable’)

​ at C:\……\themes\next\scripts\filters\comment\valine.js:11:21】

又一通搜索，找到了一个Github网友的解决方案👇：

  • Hexo 搭建博客问题#17

问题2-本地localhost查看页面变空白

解决后，继续一通配置，但发现配置后整个网站页面又空白了，一时半会找不到解决方案（包括尝试执行命令npx hexo clean），也不想继续折腾（划重点），于是，直接删除了【博客网站根目录-themes】路径下的next文件夹，重新安装NexT。

此时，又突发奇想，用NexT官方的npm命令执行npm install hexo-theme-next，发现NexT的文件夹，放到了【博客网站根目录-node_modules】路径下，文件名也变成了“hexo-theme-next”。

重新执行npx hexo clean，npx hexo server，成功！可以通过“localhost:4000/”本地访问文章了。

而执行这个过程中，发现终端命令的返回结果有了变化，多了下图中的NexT版本号、Logo等⬇️

合理怀疑git方式安装的不是最新的NexT。

不是所有配置都要一步到位配置好，一开始我是这么想的，所以一通配置搞个不停，后来发现，新手小白要有个熟悉过程，一步到位难度大，不然耗费时间、消磨意志。

⚠️更重要的是，内容比形式重要，先慢慢写起来，毕竟能坚持写下来已殊为不易！

待博客访问人数多了，再考虑评论留言、多语言切换、SEO优化等等更多客制化功能、甚至搭建服务器承载更多流量。
